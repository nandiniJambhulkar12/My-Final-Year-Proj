🎯 CONTEXT-AWARE FILTERING EXAMPLES
══════════════════════════════════════════════════════════════════════════════

Example 1: Weak Randomness Detection
───────────────────────────────────────────────────────────────────────────────
  WITHOUT CONTEXT (Old):
  ❌ random.randint(1, 6)        → Flagged (game code - FALSE POSITIVE)
  ❌ random.randint(0, 1000)     → Flagged (simulation - FALSE POSITIVE)
  ❌ token = random.randint(...) → Flagged (security - TRUE POSITIVE)

  WITH CONTEXT (New):
  ✅ random.randint(1, 6)        → NOT flagged (no "token" context)
  ✅ random.randint(0, 1000)     → NOT flagged (no "auth" context)
  ❌ token = random.randint(...) → Flagged (context found: "token")

  context = ["token", "auth", "secret", "key", "password"]
  Impact: 90% fewer false positives

───────────────────────────────────────────────────────────────────────────────
Example 2: Path Traversal Detection
───────────────────────────────────────────────────────────────────────────────
  WITHOUT CONTEXT (Old):
  ❌ with open('/etc/config.json') as f:  → Flagged (hardcoded - FP)
  ❌ path = request.args.get('file')      → Flagged (user input - TP)
       return open(path)

  WITH CONTEXT (New):
  ✅ with open('/etc/config.json') as f:  → NOT flagged (no user input)
  ❌ path = request.args.get('file')      → Flagged (context: "request")
       return open(path)

  context = ["request", "input", "file_name", "path", "user"]
  Impact: 50% fewer false positives
